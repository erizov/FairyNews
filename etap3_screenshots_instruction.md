# Скриншоты для отчёта по этапу 3 (ответы LLM по алгоритму)

Просьба приложить **скриншоты**, на которых видно, что **по алгоритму проекта** получены **ответы от LLM** (и связанные с ними результаты: структура новости, RAG, итоговый текст).

Сами изображения в репозиторий обычно **не коммитятся** — вставьте их в **Word-отчёт этапа 3** или приложите отдельным архивом по требованию преподавателя.

---

## Что должно быть видно на скриншотах

1. **Контекст запуска** (хотя бы один кадр): веб `http://127.0.0.1:8765/` **или** ноутбук `notebooks/fairy_news_final.ipynb` **или** ответ API — чтобы было понятно, откуда снято.

2. **Результат агента новостей** (косвенно): блок **сводки новости** / тем / ключевых слов на **шаге 3** веб-интерфейса (поле с итогом после генерации) **или** фрагмент JSON отчёта с полем `news_brief` (страница детального отчёта `/reports-ui/detail.html` после прогона с реальным API).

3. **Связь с RAG**: строка с **опорным источником** (`chosen_tale_source`, «Опора RAG») и при возможности **фрагмент top‑k** из детального отчёта (таблица RAG).

4. **Ответы LLM по цепочке** (что видит пользователь):
   - **текст сказки** (генерация);
   - блок **аудита** (одобрено / заметки);
   - **вопрос и эталонный ответ** (агент Q&A).

5. **Опционально** (если делаете скрин из отчёта): блок **`llm`** (`provider`, `model`) и **`heuristics`** — чтобы было видно, что прогон зафиксирован.

---

## Минимальный набор (рекомендуется)

| № | Содержание кадра |
|---|------------------|
| 1 | Шаг 1–2: выбранная новость и пресет (до генерации). |
| 2 | Шаг 3: **полный экран** с метаданными RAG, **текстом сказки**, вопросом и эталоном — **главный скриншот по LLM**. |
| 3 | Страница **детального отчёта** (`/reports-ui/detail.html?id=…`) с top‑k и эвристиками — если используете отчёты после этапа 4; иначе достаточно п. 2. |

## Важно

- Для вызова облачной модели задайте **`OPENAI_API_KEY`** (при необходимости — совместимый endpoint через `OPENAI_BASE_URL`).
- **Не вставляйте** в скриншоты ключи API и персональные данные.
